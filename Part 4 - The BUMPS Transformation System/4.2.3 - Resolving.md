# Resolving (R)

## Diagnostic Question

**“Can this problem be fixed within the current system?”**

---

## Definition

Resolving addresses a problem by **fixing it within the existing system**, restoring normal operation.

> The problem is repaired, not transformed.

---

## Essence

**Fix what’s broken.**

Resolving assumes the system is fundamentally sound.

---

## Key Characteristics

* 🔧 Repair-focused
* 🧩 Localized intervention
* ⏱️ Often repeatable
* 🏗️ System preserved

---

## Typical Signals

* “Let’s patch it”
* “We fixed the issue”
* “Back to normal”

---

## Examples

* Bug fixes
* Process tweaks
* Replacing faulty components

---

## Counter‑Plays

* **Against Revolving:** Demand measurable closure criteria.
* **Risk:** Repeated resolving may signal need for *Dissolving*.

---

## System Effect

The problem disappears temporarily but may recur if root causes remain.
