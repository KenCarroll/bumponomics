# 2.1.2 Operational Viability

The second constraint of Idealized Design is **Operational Viability**.

The system or organization should be designed so as to be completely capable of surviving in the current environment exactly as it exists today. 

Crucially, while it must be able to survive in the current environment, *it need not be immediately implementable* in the current environment. The transition path to get there may be enormously complex and require massive strategic execution (Means Planning), but the final state must be a viable, stable engine that won't instantly collapse when exposed to reality.
